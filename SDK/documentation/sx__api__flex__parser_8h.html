sx_status_t sx_api_flex_parser_transition_set ( const sx_api_handle_t  handle,
const sx_access_cmd_t  cmd,
const sx_flex_parser_header_t  from,
const sx_flex_parser_transition_t  to 
)

This API configures (enables/disables)transition between two existing flex parser headers in the parse graph. Caller must specify the header to enable the transition between "From" and "To" nodes. Depending on the "To" (next protocol field) definition, the actual size of the value used may be less than 32 bits. Note that some combination of flex parser headers may be unsupported and will return SX_STATUS_UNSUPPORTED Supported devices: Spectrum, Spectrum2, Spectrum3. Spectrum supports only

  • Fixed header types.
  • Transition from UDP to VxLAN with outer encapsulation level. Parameters:
    Parameters
    [in]handle- SX-API handle
    [in]cmd- SX_ACCESS_CMD_SET / SX_ACCESS_CMD_UNSET
    [in]from- the flex parser header to transition from(src)
    [in]to- the flex parser next transition header information(dest) Returns:
    Returns
    SX_STATUS_SUCCESS if operation completes successfully
    SX_STATUS_PARAM_ERROR if any input parameter is invalid
    SX_STATUS_ERROR general error
    SX_STATUS_UNSUPPORTED if configuration is not supported by the platform.
    SX_STATUS_NO_RESOURCES if configuration fails due to a lack of HW resources
    SX_STATUS_MODULE_UNINITIALIZED if API is called without initialization.
    SX_STATUS_CMD_UNSUPPORTED - if invalid cmd is passed

This API retrieves the configured transition information between given nodes in the parse graph. Caller can start at the root and iterate over the results to traverse the entire parse graph. Supported devices: Spectrum, Spectrum2, Spectrum3. Spectrum supports only UDP as current header. Parameters:

Parameters
[in]handle- SX-API handle
[in]curr_ph- the current flex parser header to transition from
[out]next_trans_p- a pointer to array of node transition structure. If it is NULL, the next_trans_cnt variable is filled out with numbers of transitions that could be retrieved. If the pointer is valid, node transition info is placed and next_trans_cnt takes actual number ports in array.
[out]next_trans_cnt- Specifies the number of items in the next_trans_p array

Returns:

Returns
SX_STATUS_SUCCESS if operation completes successfully
SX_STATUS_ERROR general error
SX_STATUS_UNSUPPORTED if configuration is not supported by the platform
SX_STATUS_NO_RESOURCES if configuration fails due to a lack of HW resources
SX_STATUS_MODULE_UNINITIALIZED if API called without init
SX_STATUS_PARAM_NULL if next_trans_cnt parameter is NULL

sx_status_t sx_api_flex_parser_reg_ext_point_set ( const sx_api_handle_t  handle,
const sx_access_cmd_t  cmd,
const sx_register_key_t  reg_key,
const sx_extraction_point_t *  ext_point_list_p,
const uint32_t *  ext_point_cnt_p 
)

This API sets / un-sets list of an extraction points for a given register key. Use cmd = UNSET to un-set extraction points list. Register can have up to 4 extraction points. When extracting, the last extraction point will be taken. This API together with sx_api_register_set replace sx_api_acl_custom_bytes_set, where general purpose register N represents a set of custom byte 2N and custom byte 2N+1.

Note that this API together with sx_api_register_set are mutually exclusive with sx_api_acl_custom_bytes_set, so per single SDK life cycle only one of them can be used.

Supported devices: Spectrum2, Spectrum3.

Parameters
[in]handle- SX-API handle
[in]cmd- SET / UNSET
[in]reg_key- Register key
[in]ext_point_list_p- List of extraction points
[in]ext_point_cnt_p- Number of items in ext_point_list_p (max 4)
Returns
SX_STATUS_SUCCESS if operation completes successfully
SX_STATUS_ERROR general error
SX_STATUS_UNSUPPORTED if configuration is not supported by the platform
SX_STATUS_PARAM_ERROR if reg_key is not allocated or extraction point is not valid
SX_STATUS_PARAM_NULL if ext_point_cnt_p or ext_point_list_p is NULL

This API returns the first ext_point_cnt_p extraction points of the given register key. Returned ext_point_cnt_p may be less than or equal to the requested ext_point_cnt_p. When ext_point_list_p is NULL, ext_point_cnt_p will be set with number of extraction points.

Supported devices: Spectrum2, Spectrum3.

Parameters

[in]reg_key- Register key
[in,out]ext_point_list_p- [in] If NULL: set ext_point_cnt_p with number of extraction points [out] List of extraction points
[in,out]ext_point_cnt_p- [in] Number of extraction points to get
  • [out] Number of extraction points returned
Returns
SX_STATUS_SUCCESS if operation completes successfully
SX_STATUS_ERROR general error
SX_STATUS_UNSUPPORTED if configuration is not supported by the platform
SX_STATUS_PARAM_ERROR if reg_key is not allocated or not exist
SX_STATUS_PARAM_NULL if extraction_point_cnt_p parameter is NULL
