sx_status_t sx_api_topo_device_set ( IN sx_api_handle_t  handle,
IN sx_access_cmd_t  cmd,
IN sx_topolib_dev_info_t *  dev_info 
)

This function sets the devices. Each device that is introduced to the system must be configured through the topology library. This API maintains a database that is used by other libraries to receive a device irrelevant for configuration

Supported devices: SwitchX, SwitchX2, Spectrum, Spectrum2, Spectrum3

Parameters
[in]handle- SX-API handle
[in]cmd- SX_ACCESS_CMD_ADD - Adds new device to topo lib. SX_ACCESS_CMD_DELETE - Deletes the device from topo lib. Note that the device should be removed from all forwarding tree associated with unicast/multicast/flood_tree_hndl_arr prior to that operation change. SX_ACCESS_CMD_READY - device is made ready when traffic can be routed from/to /via this device. please note that trees(unicast/multicast/flood_ must be configured with the new device
[in]dev_info- pointer to device info to set
Returns
SX_STATUS_SUCCESS if operation completes successfully
SX_STATUS_PARAM_ERROR if any input parameters is invalid
SX_STATUS_ERROR general error
SX_STATUS_PARAM_NULL if null pointer is given
